A AgentOS la base · MIT
Boilerplate · multi-frontend · multi-LLM

La base sobre la que construyes apps con IA.

AgentOS no es una app — es la plomería que cualquier app con IA necesita: cascada con failover entre proveedores LLM, multi-agente con razonamiento visible, RAG sobre tus archivos, búsqueda web, NL → SQL sobre tu base de datos, notificaciones multicanal y un file adapter universal. Tu app — dashboard, panel, integración o chat — la construyes tú; AgentOS la alimenta.

[LLM]

Cascada con circuit breaker

Una sola llamada prueba Groq → OpenRouter → Gemini. Si Groq da 429, reintenta el siguiente modelo del mismo proveedor antes de saltar; si todos los modelos fallan, marca el proveedor como caído por 60 s y prueba el próximo. Tu app nunca ve un 5xx por LLM saturado.

[AGT]

[RAG]

RAG sobre tus documentos

Sentence-aware chunking + SBERT MiniLM embeddings (384-dim) + FAISS (dev) o pgvector (prod). La pipeline es compartida entre /rag/ingest y los agentes — lo que subís se indexa en la misma instancia que el chat consulta.

[DB]

NL → SQL sobre tu DB

Conectá Postgres / MySQL / SQLite. El introspector arma el schema en texto, el LLM genera el SELECT, SafeQueryExecutor bloquea DML/DDL/multi-statement y limita filas. Devuelve el SQL + las filas para que tu UI muestre ambos.

"¿cuántos clientes > 30 años?"
▼
SELECT COUNT(*) FROM clientes WHERE edad > 30 LIMIT 200

Empezar en 5 comandos

Asumiendo Python 3.12 y Node 20+. La primera vez tarda ~3 min por sentence-transformers. Tu app real (dashboard, panel, lo que sea) vivirá fuera de este repo y consumirá la API que arranques con estos pasos. 

# 1) clonar y configurar
git clone <repo> && cd agentos
cp .env.example .env  # pega GROQ_API_KEY (gratis en console.groq.com)

# 2) instalar packages
pnpm install
pip install -e packages/llm packages/agents packages/rag packages/files services/api

# 3) backend
cd services/api && uvicorn app.main:app --reload

# 4) probar el chat-test (verifica que las capacidades responden)
pnpm dev:svelte   # http://localhost:5173

# 5) tu app real ya puede llamar a la API:
curl -X POST http://localhost:8000/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"message":"hola","language":"es"}'
